Core Capabilities

Everything You Need to
Manage Files Securely

Built with defense-in-depth architecture — seven layers of protection between your data and any threat.

🔐

AES-256 Encryption

Every file is encrypted with military-grade AES-256 before touching the disk. Keys are derived on-the-fly and never stored.

📱

Two-Factor Auth

OTP-based 2FA adds a second verification layer. 60-second expiry, 3-attempt limit, and automatic lockout protection.

🦠

Malware Scanning

Real-time signature-based scanning detects 14+ threat patterns including EICAR, shell exploits, SQL injection, and XSS.

⚡

Role-Based Access

RBAC with Admin and User roles. Granular file permissions (read, write, admin) enforce least-privilege access.

🛡️

Overflow Protection

Five-layer input validation: length limits, null-byte stripping, pattern detection, payload caps, and type blocking.

📋

Activity Logging

Every action logged with user ID and IP address. Separate threat logs track security events for forensic analysis.

How It Works

Secure by Design,
Simple to Use

From upload to download, every step is protected by multiple security layers.

01

Authenticate

Sign in with bcrypt-hashed credentials. Optional 2FA adds an OTP verification step for enhanced security. JWT tokens manage your session statelessly.

🔑
02

Upload & Scan

Drop your files — each one is scanned against 14+ malware signatures, checked for buffer overflow patterns, and validated for safe file types before storage.

📤
03

Encrypt & Store

Clean files are encrypted with AES-256 using a unique IV per file. Encrypted data is saved to disk; only metadata lives in the database.

🔒
04

Share & Control

Share files with granular permissions — read-only, read-write, or full admin. RBAC middleware enforces access on every API call.

🔗
Security Architecture

Seven Layers of
Defense-in-Depth

Layer 1

Input Validation

Length limits, regex patterns, null-byte stripping on every input field.

Layer 2

Authentication

bcrypt hashing (12 rounds), JWT tokens, optional 2FA with OTP.

Layer 3

Authorization

RBAC middleware checks role hierarchy before every protected route.

Layer 4

Encryption

AES-256-GCM (client) + AES-256-CBC (server) with unique IV per file.

Layer 5

Threat Detection

14+ malware signatures, overflow patterns, SQL injection and XSS scanning.

Layer 6

Rate Limiting

Auth endpoints capped at 10 – 20 requests/min per IP to prevent brute-force.

Layer 7

Audit Logging

20+ event types logged with timestamps and IP address for forensics.
